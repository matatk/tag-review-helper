TAG Review Helper Set-up

In order to use this extension, you need to provide a GitHub API Personal Access Token. This allows the extension to query for internal review comments that relate to a particular design review.

Important note: It's best practice to ensure that your token imparts read-only access to issues in the internal review comments repository only. Due to browser extension API limitations, the token will be stored in plain text on your local filesystem by this extension (though it is not exposed to web sites, nor other extensions, it could be leaked if your browser or system is compromised). Giving the token only the needed access helps mitigate against the risks this poses—though if you are not comfortable with the risk, you should not use this extension.

Creating a minimal token

Here's a step-by-step guide to creating a token with only the required access.

  1. Visit https://github.com/settings/personal-access-tokens/new—you will need to sign in to GitHub.

  2. Enter a name for your token—this is just for your own reference.

  3. Under "Resource owner" choose "w3ctag".

  4. Under "Expiration" choose a value such as 30, 60, or 90 days. It's strongly recommended that any tokens you create expire in a reasonable time-frame, such as a few months.

  5. Under "Repository access" choose "Only select repositories" and select the "w3ctag/design-reviews-private-brainstorming" repository.

  6. Verify that under "Overview" the first summary says "2 permissions for 1 of your repositories" and the permissions are:

    • Issues: Read-only

    • Metadata: Read-only

  7. Verify that the second summary says "0 Account permissions"

  8. Verify the expiry date.

  9. Activate the "Generate token" button.

  10. Copy the token to the clipboard, paste it into the box below, and use the "Save" button.
